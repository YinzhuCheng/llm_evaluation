## Windows 一键交付（GUI + 命令行导入）

目标：把本仓库打包成 **Windows 可执行程序（.exe）**，交给不装 Python 的用户直接运行。

---

## 你需要的输入约定（推荐）

为了让普通用户最省事，建议用 `--root`：

- 数据集：`<root>/dataset.xlsx`
- 图片目录：`<root>/images/`

GUI 里也提供了：
- **选择 root 目录**
- **从命令行导入设置**（专业用户可直接粘贴整行命令）

---

## 构建环境（只在“打包机”上需要）

以下步骤只需要你（打包者）执行一次；最终用户不需要安装 Python。

- Windows 10/11
- Python 3.11+（建议 3.12），并勾选 “Add Python to PATH”

---

## 一键打包（推荐）

在仓库根目录双击/运行：

- `packaging/windows/build.bat`

产物会在：

- `dist/EvalTool/`（文件夹版，推荐分发）

你把整个 `dist/EvalTool/` 文件夹打包成 zip 发给别人即可。

---

## 运行说明（给最终用户）

用户只需要：

1. 解压 `EvalTool.zip`
2. 双击运行 `EvalTool.exe`
3. 选择 `--root`（包含 `dataset.xlsx` 与 `images/` 的上一级目录）
4. 填好模型/裁判配置（或用“从命令行导入设置”粘贴整行命令）
5. 点击运行

---

## 常见问题

### 1) 为什么不能用 exe 再去启动 python 子进程？
本项目 GUI 在打包模式下已改为“进程内直接调用评测逻辑”，避免 PyInstaller 环境下无法可靠找到 `python.exe` 的问题。

### 2) 点击“停止”能立刻停吗？
停止是“尽力取消”：会阻止后续请求继续发起，但已发出的网络请求可能需要等待返回/超时。
